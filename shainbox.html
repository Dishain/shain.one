§ 01 · Architecture

Files, not a database.
Your library is a folder you can open, back up, or move - with or without this app.

01 / PORTABLE

Point it at Drive. It follows.

The root folder and its config move together. Open on a new machine, select the folder, and your boards, tags, favourites, and smart filters are all there.

02 / REBUILDABLE

Cache is disposable. Sidecars are truth.

Thumbnails and the search index live in cache/ and can be nuked at any time. Your actual work - tags, notes, favourites - lives in small sidecar files next to your images.

03 / OPEN

Plain formats. No vendor lock.

Images stay as images. Videos stay as videos. Metadata is JSON. If this app disappears tomorrow, you still have a clean library of files in folders.

§ 03 · Features

Built for capture, not collection management. Every pixel earns its place.

F / 01

Stored as folders

Top-level folders are boards. Drop files in from Finder, Figma, or a browser - they show up in ShainBox instantly. Move them out and they keep working.

FINDER ←→ SHAINBOX  ·  Always in sync
F / 02

Masonry library

A dense, scrollable grid that respects the shape of each reference. No crops, no forced squares. Images, videos, and MP4 GIFs - side by side.

PNG · JPG · GIF · MP4 · WEBP  ·  Native
F / 03

Fast capture

Global shortcut to stash a file from the clipboard or a drop zone. Automatic deduplication via content hash - the same reference never gets saved twice.

xxHash3  ·  Dedup before write
F / 04

Keyboard-first

Browse with J / K. Preview with Space. Tag with T. No mouse required.
